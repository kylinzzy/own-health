# 健康瘦身小程序 - 部署说明

## 一、环境准备

### 1.1 注册微信小程序账号
1. 访问 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击"立即注册"，选择"小程序"
3. 填写邮箱、密码等信息完成注册
4. 进行主体认证（个人开发者选择个人主体）

### 1.2 下载微信开发者工具
1. 访问 [微信开发者工具下载页](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 下载对应系统版本的开发者工具
3. 安装并登录（使用小程序账号扫码登录）

## 二、项目导入

### 2.1 获取项目代码
将本小程序工程文件夹 `health-miniapp` 复制到本地工作目录。

### 2.2 修改项目配置
打开 `project.config.json` 文件，修改 `appid` 为你自己的小程序 AppID：

```json
{
  "appid": "wx YOUR_APPID_HERE"
}
```

> AppID 可在微信公众平台 -> 开发 -> 开发设置 中获取

### 2.3 导入项目
1. 打开微信开发者工具
2. 点击"导入项目"
3. 选择 `health-miniapp` 文件夹
4. 输入你的 AppID
5. 点击"确定"导入

## 三、预览调试

### 3.1 编译运行
导入成功后，开发者工具会自动编译项目。如无报错，即可在模拟器中预览小程序。

### 3.2 真机调试
1. 点击开发者工具顶部的"真机调试"
2. 使用微信扫码，在手机上预览
3. 可开启"调试模式"查看控制台日志

### 3.3 常见问题

| 问题 | 解决方案 |
|------|----------|
| 编译报错 "app.json 不存在" | 确认导入的是 `health-miniapp` 文件夹，而非外层目录 |
| 页面显示空白 | 检查 `app.json` 中 pages 配置是否正确 |
| 数据不保存 | 检查微信开发者工具 "详情" -> "本地设置" -> "不校验合法域名" 是否勾选 |
| 图表不显示 | 确保在真机或模拟器中查看，canvas 需要渲染环境 |

## 五、发布上线

### 5.1 上传代码
1. 在微信开发者工具中点击右上角"上传"
2. 填写版本号（如 1.0.0）和项目备注
3. 点击"上传"

### 5.2 提交审核
1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入"管理" -> "版本管理"
3. 在"开发版本"中找到上传的版本
4. 点击"提交审核"
5. 填写小程序功能介绍和类目信息

### 5.3 审核注意事项
- 本小程序为纯工具类，不涉及用户登录和支付
- 所有数据本地存储，不上传服务器
- 审核时说明小程序用途：个人健康管理工具

### 5.4 发布上线
审核通过后，在"版本管理"中点击"发布"即可上线。

## 六、项目结构说明

```
health-miniapp/
├── app.js                 # 应用入口，初始化数据
├── app.json               # 全局页面路由和配置
├── app.wxss               # 全局样式和CSS变量
├── sitemap.json           # 微信搜索索引
├── project.config.json    # 项目配置（需修改appid）
├── pages/                 # 页面目录
│   ├── index/             # 首页-数据概览
│   ├── guide/             # 膳食指南
│   ├── guide-detail/      # 指南详情
│   ├── recipes/           # 减脂食谱库
│   ├── recipe-detail/     # 食谱详情
│   ├── weight/            # 体重管理
│   ├── weight-record/     # 体重录入
│   ├── diet/              # 饮食记录
│   ├── diet-record/       # 饮食录入
│   ├── exercise/          # 运动记录
│   ├── exercise-record/   # 运动录入
│   └── analysis/          # 健康分析
├── components/            # 公共组件（预留）
├── utils/                 # 工具函数
│   ├── storage.js         # 本地存储封装
│   ├── date.js            # 日期处理
│   ├── chart.js           # 图表绘制
│   ├── analysis.js        # 健康分析算法
│   └── constants.js       # 常量定义
├── data/                  # 静态数据
│   ├── guide-data.js      # 膳食指南数据
│   └── recipe-data.js     # 食谱初始数据
└── docs/                  # 文档
    ├── ARCHITECTURE.md    # 架构设计
    └── DEPLOY.md          # 部署说明
```

## 七、后续自定义

### 7.1 修改目标体重
在"体重管理"页面，目标体重可在代码中修改，或后续添加设置页面。

### 7.2 添加更多食谱
编辑 `data/recipe-data.js` 文件，按现有格式添加新食谱。

### 7.3 修改配色
编辑 `app.wxss` 中的 CSS 变量，调整主色、辅色等。

### 7.4 数据备份
由于数据存储在本地，如需备份可：
1. 在开发者工具中打开"调试器"
2. 在 Console 中执行 `wx.getStorageInfoSync()` 查看存储
3. 使用 `wx.getStorageSync(key)` 导出数据

## 八、技术说明

### 8.1 数据存储
- 使用 `wx.storage` 进行本地持久化存储
- 所有数据保存在用户手机本地，不上传服务器
- 存储容量限制约 10MB

### 8.2 图表实现
- 使用原生 Canvas API 自研轻量图表
- 支持体重趋势曲线绘制
- 无第三方图表库依赖

### 8.3 兼容性
- 基础库版本要求：2.32.0+
- 支持 iOS 和 Android 微信
- 适配主流手机屏幕尺寸
